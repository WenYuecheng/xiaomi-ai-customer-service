## 一、项目背景与目标

**项目名称**：小米智能客服机器人（课程验收级 RAG 智能客服平台）

**核心目标**：构建一个面向课程验收的可信 RAG（检索增强生成）客服系统，能够基于真实小米产品文档回答用户问题，并完整展示 AI 执行轨迹，满足教学验收的可解释性要求。

**系统定位**：前后端分离的全栈 Web 应用，后端以 Python FastAPI 实现，前端以 Vue 3 实现，通过 Docker Compose 一键部署。

## 四、五阶段 AI 执行轨迹（核心验收点）

系统完整实现了可视化的 AI 执行链路，每次问答均在前端卡片中展开展示：

```
DeepSeek 1/3 问题理解（意图分类 + 问题改写）
  -> BGE 召回 8 个候选片段（向量相似度检索）
  -> DeepSeek 2/3 AI 重排（保留最多 4 个高相关片段）
  -> DeepSeek 3/3 依据来源生成回答（流式 SSE）
  -> Citation Guard 引用真实性校验
```

## 六、安全边界（必须做到要求）

- JWT 仅接受 HS256 算法，验证签名、签发方、有效期
- 注册限流：同一来源 10 分钟最多 5 次
- 注册固定为普通角色，不接受 role 字段
- 密码 Bcrypt 加密，密钥只从环境变量读取
- 密码修改后 token_version 递增，旧 JWT 即时失效
- 上传文件：格式/签名/大小/路径四重校验
- 文档内容不作为代码或系统命令执行
- 敏感词（支付密码、银行卡密码）触发拦截并审计
- Mock 订单明确标识，不含真实个人信息
- 用户画像只统计产品/意图/反馈，不保存地址/电话/支付

## 八、当前遗留限制

| 限制 | 说明 |
|------|------|
| 订单/物流 | 为 Mock 演示数据，未接入真实小米订单系统 |
| 支付/退款/售后 | 需要官方接口授权，当前不支持 |
| 多渠道 | 不支持 APP/微信等渠道接入 |
| 实时价格 | 价格仅来自知识文档，标注采集时间 |
| 生产密钥 | 需手动配置 JWT_SECRET（不少于 32 字符）和 LLM 密钥 |
