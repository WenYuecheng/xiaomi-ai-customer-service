# 对话消息左右分栏设计

## 目标

修正可信问答页面的消息方向：用户消息整体靠右，小爱客服消息整体靠左。身份标识、名称和气泡必须作为一个视觉单元随角色对齐，不能再出现用户头像留在左侧而气泡单独靠右的情况。

## 方案

继续复用 `ChatMessage` 单一组件，通过 `message--user` 和 `message--assistant` 角色修饰类控制网格与对齐，不复制两套模板。

- 小爱客服：左侧显示头像和“小爱客服”，右侧显示回答、AI 执行轨迹、来源、反馈和工单操作。
- 用户：左侧保留弹性空白，右侧先显示消息气泡，再显示头像和“你”；气泡内容右对齐。
- 桌面端限制用户气泡宽度，短问题保持紧凑，长问题可自然换行。
- 窄屏下仍保持用户消息靠右、小爱客服靠左；身份标识不与正文争抢一整行宽度。
- 保留键盘可访问性、减少动态效果设置以及所有现有交互行为。

## 实现边界

仅修改消息布局样式及对应组件测试，不修改消息数据结构、SSE 流程、Markdown 渲染、AI 执行轨迹、来源、反馈或工单接口。

## 测试与验收

- 组件测试断言用户与客服消息具有正确角色类和身份内容。
- 浏览器验证桌面端用户身份位于右侧、客服身份位于左侧。
- 浏览器验证窄屏布局没有溢出或身份错位。
- 运行全部前端 Vitest、Vue TypeScript 检查和 Vite 生产构建。
