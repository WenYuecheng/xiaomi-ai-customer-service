# 小米智能客服机器人设计说明

## 目标

建设一个单人可完成、可测试、可通过 Docker Compose 演示的智能客服系统。范围以 Jira CSV 中 32 条 Story，加上语义预处理、用户画像、热词、历史会话压缩、混合推荐和离线模型训练为准。

## 架构

- 前后端分离，后端为 FastAPI 模块化单体，前端为 Vue 3 单页应用。
- SQLite 保存业务记录和可审计的文本块；Chroma 仅保存向量和检索元数据；原始上传文件独立持久化。
- 文档任务使用单进程轻量 worker，状态为 `queued/running/succeeded/failed/cancelled`，步骤幂等且失败可重试。
- 模型接口支持 Mock、OpenAI-compatible 与 Ollama；默认 Mock 保证无密钥测试和演示。
- 知识答案只能使用实际检索上下文；每条事实型回答必须有来源，否则进入兜底。

## 核心数据流

创建知识库 → 安全上传 → 解析与清洗 → 结构感知切分 → 向量化 → 混合检索 → 置信度判断 → 基于证据生成或兜底 → JSON/SSE 返回 → 保存会话、来源、日志和行为事件。

## 安全边界

- JWT 必须验证签名和算法，密码使用 Bcrypt，密钥仅从环境变量读取。
- 上传文件采用扩展名、MIME、文件签名和大小校验；服务端生成存储名并强制目录边界。
- 不执行文档内容、模型输出或用户输入中的代码、命令和工具指令。
- Mock 订单不包含真实个人信息，画像仅使用产品偏好和交互信号。

## 验收

- 四类文档均可入库、预览、重建和删除。
- 30 问回归正确率不低于 80%，知识问答 100% 有真实引用或正确兜底。
- 同一会话支持最近 10 轮和滚动摘要，不同用户与会话完全隔离。
- Docker 演示连续三次无阻断错误，持久化数据在重启后保留。
